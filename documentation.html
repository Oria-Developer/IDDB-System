Technical Reference

Simplified documentation for integrating the IDDB Python library into your bot infrastructure.

01. Installation

Install the official package from PyPI:

pip install iddb-system==0.1.0

02. Initialization

Set up the client within your bot's constructor to handle background tasks.

from iddb.core import IDDBClient

# Inside your discord.Client or commands.Bot class
self.iddb = IDDBClient(client=self, channel_id=1465966978618495028)

async def setup_hook(self):
    await self.iddb.start()

03. Methods

.log()

Pushes data to the queue for asynchronous logging.

await bot.iddb.log(guild_id: int, action_type: str, metadata: dict)
guild_id Target server ID (integer). Use 0 for global logs.
action_type Category identifier (e.g. "USER_JOIN").
metadata Dictionary containing your custom JSON data.

04. Internal Architecture

Rate Limiting

The SDK enforces a 2.5s delay between writes to ensure Discord API safety. High-frequency logs are automatically queued.

Fragmentation

Large metadata dictionaries exceeding 1800 characters are split into multiple ordered messages and linked via a unique Action ID.
